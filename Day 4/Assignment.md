# Assignment – Day 4

## Scenario

You are an **Odoo Functional Consultant**.

A new employee named **John Sale** has joined **ABC Group** as a **Sales Executive**.

The company has two businesses:

- Ravinder Technologies Pvt Ltd
- Ravinder Retail Pvt Ltd

Your responsibility is to create the user, assign the correct permissions, and ensure the employee can work only with the required modules.

Answer the following questions.

---

# 1. What is a User in Odoo?

**Answer:**

A User is a person who can log in to Odoo and perform business operations based on the permissions assigned.

Examples:

- Administrator
- Sales Manager
- Sales Executive
- Accountant
- Inventory Manager
- HR Officer

Each user has a unique login account.

---

# 2. How do you create a new user?

**Answer:**

Steps:

1. Open **Settings**.
2. Go to **Users & Companies**.
3. Click **Users**.
4. Click **New**.
5. Enter:
   - Name
   - Email
   - Login
   - Password
6. Click **Save**.

The user can now log in to Odoo.

---

# 3. What are Access Rights?

**Answer:**

Access Rights determine what actions a user can perform in Odoo.

Permissions include:

- Read
- Write
- Create
- Delete

These permissions help protect business data and prevent unauthorized access.

---

# 4. What are Security Groups?

**Answer:**

Security Groups define which modules and features a user can access.

Examples:

- Sales User
- Sales Administrator
- Inventory User
- HR Administrator
- Accounting User

A user can belong to one or more security groups depending on their job role.

---

# 5. What are Record Rules?

**Answer:**

Record Rules control which records a user can view or modify.

Example:

John Sale can view only his own quotations if assigned the **Sales / User: Own Documents Only** role.

A Sales Manager can view quotations created by all sales executives.

This improves data security and privacy.

---

# 6. Why are Access Rights important?

**Answer:**

Access Rights help to:

- Protect business information
- Prevent unauthorized changes
- Control user responsibilities
- Improve system security
- Reduce operational errors

Only authorized users can perform specific actions.

---

# 7. How can a user switch between companies?

**Answer:**

Steps:

1. Click the company name in the top-right corner.
2. Select the required company.
3. Continue working in the selected company.

No logout is required.

This feature is useful for organizations managing multiple companies.

---

# 8. What is the difference between Groups, Access Rights, and Record Rules?

**Answer:**

| Feature | Purpose |
|----------|---------|
| **Groups** | Determines which modules and features a user can access. |
| **Access Rights** | Defines whether the user can Read, Write, Create, or Delete records. |
| **Record Rules** | Controls which specific records the user is allowed to view or edit. |

Together, these three features provide complete security and access control in Odoo.

---

# 9. What are the responsibilities of an Odoo Functional Consultant?

**Answer:**

An Odoo Functional Consultant should:

- Create and manage users.
- Assign appropriate security groups.
- Configure access rights.
- Apply record rules.
- Test user permissions.
- Train employees to use Odoo.
- Ensure business data remains secure.

---

# Conclusion

As an **Odoo Functional Consultant**, understanding **Users, Roles, Access Rights, Security Groups, and Record Rules** is essential for implementing a secure and efficient Odoo system.

Proper user management ensures that employees can access only the information and features required for their job while protecting sensitive business data.
