# Day 2 – Odoo Interface & Navigation

## Learning Objectives

By the end of Day 2, you will be able to:

- Navigate the Odoo interface
- Understand the Apps Dashboard
- Use the App Switcher
- Understand Menus and Submenus
- Search records
- Apply Filters
- Use Group By
- Save Favorites
- Understand different Views
- Configure User Preferences

---

# Odoo User Interface

The Odoo interface is designed to make business management simple and user-friendly.

Main Components:

1. Apps Dashboard
2. Top Navigation Bar
3. Search Bar
4. Menu
5. Submenu
6. Action Buttons
7. User Menu

---

# Apps Dashboard

The Apps Dashboard displays all installed business applications.

Examples:

- CRM
- Sales
- Purchase
- Inventory
- Accounting
- HR
- Manufacturing

Each app represents a business department.

---

# Top Navigation Bar

The top bar contains:

- App Name
- Search
- Notifications
- Activities
- Settings
- User Profile

---

# App Switcher

The App Switcher allows users to quickly move between different modules.

Example:

CRM → Sales → Inventory → Accounting

No need to log out.

---

# Menu

Each module contains menus.

Example:

Sales

↓

Orders

↓

Quotations

↓

Sales Orders

Menus organize business activities.

---

# Search Bar

Used to find records quickly.

Example:

Search Customer

"ABC Traders"

Result:

Customer Record

---

# Filters

Filters display specific records.

Examples:

- My Quotations
- Draft Orders
- Confirmed Orders

Benefits:

- Saves time
- Quick analysis

---

# Group By

Groups records.

Example:

Sales Orders

↓

Group By

↓

Salesperson

Result:

Orders grouped by each salesperson.

---

# Favorites

Save frequently used filters.

Example:

"My Customers"

Click once to view.

---

# Odoo Views

Odoo supports multiple views.

List View

Shows records in table format.

Form View

Shows complete record details.

Kanban View

Card-based layout.

Calendar View

Displays meetings and schedules.

Activity View

Shows pending activities.

Pivot View

Business analysis.

Graph View

Charts and visual reports.

---

# User Preferences

Users can configure:

- Language
- Time Zone
- Signature
- Password
- Notifications

---

# Multi-Company

Odoo supports multiple companies.

Example:

ABC Supermarket

XYZ Textiles

Same user can switch between companies.

---

# Notifications

Odoo displays:

- Messages
- Tasks
- Activities
- Approval Requests

---

# Activities

Activities remind users about:

- Calls
- Meetings
- Follow-ups
- Deadlines

---

# Functional Consultant Responsibility

A Functional Consultant should know:

- How users navigate Odoo
- Which views are suitable
- How departments use menus
- How to explain navigation to end users
